## 2. Dataset Description

| Property | Value |
|----------|-------|
| Source | Google Drive (provided) |
| Format | `.npy` file (NumPy array, pickle-serialized) |
| Total Samples | 409 movie scene images |
| Annotation Type | Bounding box coordinates (normalized x, y) |
| Labels | `['Face']` per annotation |
| Image Sizes | Variable (180px–4016px height, 182px–6016px width) |
| Channels | Mostly RGB (3ch), 1 grayscale, some RGBA (4ch) |
| Faces per Image | 1–16 (majority 1–3) |

### Data Structure
Each sample in the `.npy` file is a pair: `[image, annotations]`
- `image`: NumPy array (H, W, 3) — the movie scene screenshot
- `annotations`: List of dicts, each containing:
  - `label`: `['Face']`
  - `points`: 2 normalized coordinates `[{x, y}, {x, y}]` (top-left, bottom-right)
  - `imageWidth`, `imageHeight`: Original dimensions

### Key Data Insights
- Images have highly variable dimensions requiring standardized resizing
- Annotations are bounding boxes, not pixel-level masks — binary masks are generated by filling bounding box regions
- Face regions are small relative to image area (class imbalance) — motivates use of Dice Loss
- Mixed channel formats require standardization to 3-channel RGB

## 3. Preprocessing Pipeline

### Steps
1. **Channel Standardization**: Grayscale → RGB, RGBA → RGB
2. **Mask Generation**: Convert bounding box annotations to binary masks (fill rectangles)
3. **Resize**: All images and masks → 256×256 (bilinear for images, nearest-neighbor for masks)
4. **Normalization**: Pixel values scaled to [0, 1] (float32)
5. **Train/Val Split**: 80/20 stratified split (327 train, 82 validation)

### Data Augmentation (Online)
Applied randomly each epoch via custom `DataGenerator`:

| Augmentation | Probability | Parameters |
|-------------|-------------|------------|
| Horizontal Flip | 50% | Mirror left-right |
| Rotation | 50% | ±15 degrees, reflect border |
| Brightness | 50% | Factor 0.7–1.3 |
| Gaussian Blur | 30% | Kernel 3×3 or 5×5 |

All augmentations are applied identically to both image and mask to maintain alignment.

## 4. Model Architecture

### U-Net with MobileNetV2 Encoder

```
Input (256×256×3)
    │
    ▼
┌─────────────────────────────┐
│  MobileNetV2 Encoder        │  (Pretrained on ImageNet)
│  (Frozen / Partially frozen)│
│                             │
│  Skip Connections:          │
│  ├─ block_1_expand_relu     │  128×128×96
│  ├─ block_3_expand_relu     │   64×64×144
│  ├─ block_6_expand_relu     │   32×32×192
│  ├─ block_13_expand_relu    │   16×16×576
│  └─ block_16_project        │     8×8×320  (bottleneck)
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  Custom Decoder             │
│                             │
│  Each block:                │
│  UpSampling2D(2×2)          │
│  → Concatenate(skip)        │
│  → SeparableConv2D + ReLU   │
│  → BatchNormalization       │
│  → SpatialDropout2D(0.2)    │
│  → SeparableConv2D + ReLU   │
│  → BatchNormalization       │
│                             │
│  8→16→32→64→128→256         │
└─────────────────────────────┘
    │
    ▼
Conv2D(1, 1×1, sigmoid)
    │
    ▼
Output (256×256×1) — Binary face mask
```

### Design Decisions
- **SeparableConv2D** instead of Conv2D: Fewer parameters, reduces overfitting on small dataset
- **SpatialDropout2D(0.2)**: Drops entire feature maps, better regularization for spatial data than standard Dropout
- **Skip Connections**: Preserve spatial detail from encoder at multiple resolutions
- **Sigmoid Output**: Binary classification per pixel (face vs background)

## 5. Custom Loss Functions & Metrics

### Dice Coefficient (Metric)
Measures overlap between predicted and ground truth masks. Range [0, 1].

```python
def dice_coefficient(y_true, y_pred, smooth=1e-6):
    y_true_f = keras.ops.cast(keras.ops.reshape(y_true, [-1]), 'float32')
    y_pred_f = keras.ops.cast(keras.ops.reshape(y_pred, [-1]), 'float32')
    inter = keras.ops.sum(y_true_f * y_pred_f)
    return (2.*inter+smooth) / (keras.ops.sum(y_true_f)+keras.ops.sum(y_pred_f)+smooth)
```

### Dice Loss
```python
def dice_loss(y_true, y_pred):
    return 1.0 - dice_coefficient(y_true, y_pred)
```

### BCE + Dice Loss (Primary)
Combined loss providing stable gradients (BCE) and overlap optimization (Dice).
```python
def bce_dice_loss(y_true, y_pred):
    bce = keras.ops.mean(keras.losses.binary_crossentropy(y_true, y_pred))
    return bce + dice_loss(y_true, y_pred)
```

### Focal + Dice Loss (Experiment)
Handles class imbalance by down-weighting easy examples.
```python
def focal_loss(y_true, y_pred, alpha=0.25, gamma=2.0):
    y_pred = keras.ops.clip(y_pred, 1e-7, 1.0 - 1e-7)
    bce = -(y_true * keras.ops.log(y_pred) + (1-y_true) * keras.ops.log(1-y_pred))
    p_t = y_true * y_pred + (1-y_true) * (1-y_pred)
    return keras.ops.mean(alpha * (1-p_t)**gamma * bce)
```

### IoU Metric
```python
def iou_metric(y_true, y_pred, smooth=1e-6):
    y_true_f = keras.ops.cast(keras.ops.reshape(y_true, [-1]), 'float32')
    y_pred_f = keras.ops.cast(keras.ops.reshape(y_pred > 0.5, [-1]), 'float32')
    inter = keras.ops.sum(y_true_f * y_pred_f)
    union = keras.ops.sum(y_true_f) + keras.ops.sum(y_pred_f) - inter
    return (inter+smooth) / (union+smooth)
```

## 6. Training Strategy

### Phase 1: Frozen Encoder
| Parameter | Value |
|-----------|-------|
| Encoder | Fully frozen (MobileNetV2 ImageNet weights) |
| Optimizer | Adam, lr=1e-3 |
| Loss | BCE + Dice |
| Batch Size | 8 |
| Max Epochs | 80 |
| Early Stopping | patience=20, monitor=val_dice_coefficient |
| LR Scheduler | ReduceLROnPlateau, factor=0.5, patience=8 |
| Augmentation | Online (random each epoch) |

**Result**: Val Dice = 0.71, trained for 50 epochs (early stopped at 30 best)

### Phase 2: Fine-tuning
| Parameter | Value |
|-----------|-------|
| Encoder | Blocks 14-16 unfrozen |
| Optimizer | Adam, lr=1e-5 |
| Max Epochs | 40 |
| Early Stopping | patience=15 |

**Result**: Did not improve over Phase 1. Best model remains Phase 1 checkpoint.

### Why Phase 2 Didn't Help
- Very low LR (1e-5) with only 3 unfrozen blocks provided minimal adaptation
- The frozen encoder features from ImageNet were already sufficient for this task
- Small dataset (327 training images) limits fine-tuning benefit

## 7. Hyperparameter Experiments

### Experiment Setup
Each experiment trains a fresh model for 30 epochs with early stopping (patience=10).

### Results

| Experiment | Optimizer | LR | Batch | Loss | Dice | IoU | F1 | Speed (ms) |
|-----------|-----------|-----|-------|------|------|-----|-----|------------|
| Baseline | Adam | 1e-3 | 8 | BCE+Dice | 0.7095 | 0.5765 | 0.7095 | 93.6 |
| **SGD_momentum** | **SGD(m=0.9)** | **1e-2** | **8** | **BCE+Dice** | **0.7425** | **0.5905** | **0.7425** | **84.9** |
| Adam_lr1e-4 | Adam | 1e-4 | 8 | BCE+Dice | 0.2480 | 0.1416 | 0.2480 | 81.9 |
| Adam_batch16 | Adam | 1e-3 | 16 | BCE+Dice | 0.0000 | 0.0000 | 0.0000 | 84.0 |
| Focal_Dice | Adam | 1e-3 | 8 | Focal+Dice | 0.7358 | 0.5821 | 0.7358 | 83.5 |
| Pure_Dice | Adam | 1e-3 | 8 | Dice | 0.7354 | 0.5815 | 0.7354 | 89.0 |
| AdamW | AdamW(wd=1e-4) | 1e-3 | 8 | BCE+Dice | 0.7374 | 0.5841 | 0.7374 | 110.0 |

### Observations

1. **SGD with Momentum (Best — Dice 0.7425)**: Noisier gradients from SGD helped generalization on this small dataset. Higher initial LR (1e-2) with momentum (0.9) allowed faster convergence while the noise prevented overfitting.

2. **Adam lr=1e-4 (Dice 0.2480)**: Learning rate too low for a frozen encoder. The decoder couldn't learn meaningful features in 30 epochs. Would need significantly more epochs or a warmup schedule.

3. **Batch Size 16 (Dice 0.0000)**: Complete failure. With 327 training images and batch size 16, only ~20 gradient updates per epoch. The model never converged. Small datasets require small batch sizes for sufficient gradient updates.

4. **Focal + Dice (Dice 0.7358)**: Comparable to baseline. Achieved highest recall (0.8240) by focusing on hard-to-classify boundary pixels. Useful when detecting all faces matters more than precision.

5. **Pure Dice (Dice 0.7354)**: Similar to BCE+Dice. Directly optimizing the target metric works but lacks the gradient stability that BCE provides in early training epochs.

6. **AdamW (Dice 0.7374)**: Weight decay didn't significantly help. The main bottleneck is dataset size and annotation quality (bounding boxes vs pixel masks), not model overfitting.

### Key Insight
All working models cluster around Dice 0.73–0.74. This ceiling is primarily due to:
- Bounding-box annotations (not pixel-perfect face masks)
- Small dataset size (409 images)
- Pixel-level annotations and more data would significantly improve scores

## 8. Evaluation Metrics

### Metrics vs Targets

| Metric | Achieved | Target | Status |
|--------|----------|--------|--------|
| Dice Coefficient | 0.7425 | >0.92 | Below target |
| IoU | 0.5905 | >0.88 | Below target |
| F1-Score | 0.7425 | >0.90 | Below target |
| Inference Speed | ~85ms | <100ms | ✅ Met |

### Why Targets Were Not Fully Met
- **Annotation quality**: Ground truth is bounding boxes, not pixel-level masks. Even a perfect model predicting exact bounding boxes would have Dice < 1.0 due to non-face pixels inside the box.
- **Dataset size**: 409 images is small for training a segmentation model. Standard face segmentation datasets (CelebAMask-HQ, LFW) have 10,000+ images.
- **Inference speed target met**: The model runs under 100ms on GPU, meeting real-time requirements.

### Evaluation Functions Used
- **Dice Coefficient**: Custom Keras metric (see Section 5)
- **IoU**: Custom Keras metric (see Section 5)
- **F1, Precision, Recall**: scikit-learn `f1_score`, `precision_score`, `recall_score` on flattened binary masks
- **Inference Speed**: Average of 20 predictions (excluding 5 warmup runs)

## 13. Known Limitations & Future Improvements

### Current Limitations
- Dice score (0.74) below target (0.92) due to bounding-box annotations and small dataset
- Model predicts rectangular face regions (influenced by bounding-box training data)
- No face recognition (only detection/segmentation)
- Single-class segmentation (face vs background only)

### Potential Improvements
- Use pixel-level face mask annotations (e.g., CelebAMask-HQ dataset)
- Increase dataset size with additional movie scene images
- Try attention mechanisms (Attention U-Net) for better boundary detection
- Implement test-time augmentation (TTA) for improved predictions
- Add face recognition pipeline post-segmentation for cast identification
- Deploy on cloud (Hugging Face Spaces, AWS, Azure) for public access
- Add webcam real-time detection support in Streamlit app
